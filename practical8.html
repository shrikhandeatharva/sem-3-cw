AUTOMATIC STREEET LIGHT CONTROL

OBJECTIVE AND GOALS

  • This project aims to design an automatic street light that works in both conventional as well as non-conventional energy resources.
  • Using LDR we control the street light, when the LDR value falls above the threshold value the lights are switched on and when the value falls below the threshold value the lights are switched off.
  • The main goal is to save eletricity. Because many a times, the street lights are on even in the morning when they are of no lot of energy can be saved if we use this mechanism.we can save the man power, as well as reduce wastage of electricity.

SOFTWARE USED

  1. uvision IDE (Keil) - For ARM
  2. Proteus - simulation tool

THE OROTICAL THINKING

During day time there is sunlight so we don't need any street lighting but during night time there's no lighting so we need street lights. so to make this more convenient we can design a project using hardware and software in which street lights turn on and off according to time on its own, also called an automatic street lighting project. In order to save and conserve energy in an efficient way we are designing this project. so there are two conditions in this project,one is day time and other is night time. Day time means full lighting and lights should be off. Night time means no light and lights should turn on automatically, so the hardware building blocks required for this project are sensors that are LDR (Light dependent resistors) as input devices and street lights are output devices which interface with microcontrollers. During day time Lol will offer very less resistance and during night time LDR will offer high resistance.

  • Day time (presence of 1ight) ---> LDR less resistance Lights turn off
  • ight time(absence of light) ---> LDR high
  • Input device ---> sensors LDR (Light Dependent Resistor)
  • output device ---> street lights

Light sensor
